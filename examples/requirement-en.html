Add a usage-based price calculator to the pricing page

This document specifies a calculator on the pricing page: visitors set an expected usage amount and see the estimated cost. It exists because a usage-based plan has no fixed price to list in the plan table.

StatusDraft Version0.1

Background

The pricing page lists plan tiers with fixed monthly prices. The usage-based plan bills by consumption, so the question “what would I pay” has no answer on the page today.

The calculator keeps the answer on the page: a visitor estimates their bill before starting a trial or talking to sales.

Scope

  • In scope: the calculator module on the public pricing page, covering the usage input, the billing-period switch, and the estimated price display.
  • Out of scope: checkout, invoicing, usage metering, and changes to the plan tiers.

Behavior

  1. The calculator sits directly below the plan comparison table on the pricing page.
  2. A slider sets the expected usage amount, and a readout next to it shows the current value.
  3. A two-option segmented switch selects monthly or annual billing.
  4. The result area shows the estimated total for the selected period and updates on every input change, without a page reload and without a network request.
  5. The estimate equals the published unit price multiplied by the selected usage, rounded to two decimals.

Decision: client-side calculation

The estimate is computed in the browser from the published unit prices. The calculator keeps working on a slow or interrupted connection.

Interaction details

The usage slider below sets the expected amount in thousand requests per month: the fill tracks the thumb, and the readout updates on every input event while dragging, not only on release. Arrow keys move the value in single steps.

The billing-period switch below chooses monthly or annual billing: options are mutually exclusive, the selected option gets the raised knob, and a status line confirms the selection. Switching the period never resets the usage value.

Edge cases

  • If the selected usage exceeds the range covered by published pricing, the result area shows a prompt to contact sales instead of a number.
  • If pricing data fails to load, the calculator shows an error message with a retry action; the rest of the pricing page keeps working.
  • At the minimum usage step, the estimate shows the base fee of the usage-based plan.
  • Switching from annual back to monthly preserves the current usage value.

Acceptance criteria

  • Dragging the slider updates the readout and the estimate on every input event, not only on release.
  • Switching between monthly and annual recalculates the displayed total and preserves the usage value.
  • The displayed estimate equals the published unit price times the selected usage, rounded to two decimals.
  • With the network disabled after page load, the calculator still responds to slider and switch input.
  • When pricing data fails to load, the calculator shows an error with a retry action, and the plan comparison table still renders.
  • Selecting a usage above the published range shows the contact-sales prompt instead of an estimate.
